# 首次登录与项目初始化

## 登录 Codex

安装完成后，先运行：

```powershell
codex --version
codex login
```

登录方式可能随官方版本更新而变化，常见方式包括：

- 打开浏览器完成 ChatGPT / OpenAI 登录。
- 在终端中按提示复制设备码。
- 使用 API Key 作为认证方式。

如果浏览器能打开但 CLI 收不到登录结果，通常是回调地址、代理或本地端口被拦截。可以按命令行提示尝试备用登录流程。

## 使用 API Key

如果使用 API Key，建议通过环境变量提供，不要写入代码仓库：

```powershell
$env:OPENAI_API_KEY = "sk-..."
```

用户级长期设置：

```powershell
[Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "sk-...", "User")
```

取消：

```powershell
[Environment]::SetEnvironmentVariable("OPENAI_API_KEY", $null, "User")
```

注意：

- 不要把真实 key 写进 README、脚本或截图。
- 如果 key 泄露，应立即在控制台撤销并重新生成。
- 公司项目建议使用团队统一管理的 key 或企业账号策略。

## 在项目目录启动

进入一个 Git 项目目录：

```powershell
cd D:\program\project\your_project
git status
codex
```

建议启动前先确认工作区状态：

```powershell
git status --short
```

如果已有未提交改动，告诉 Codex 哪些文件是你正在改的，哪些不能碰。

## 第一次对话建议

可以先让 Codex 只读不改：

```text
请先阅读这个项目的 README 和目录结构，概括项目用途、主要模块和启动方式。
暂时不要修改任何文件。
```

然后再给一个低风险任务：

```text
请帮我检查 README 里是否缺少安装步骤。
如果需要修改，只改 README.md。
改完后总结变更，并告诉我是否需要运行测试。
```

## 权限确认

Codex 在本地执行命令时，可能遇到权限确认。建议团队建立统一规则：

- 允许读取文件和运行只读检查命令。
- 允许运行测试、lint、format。
- 修改文件前要求说明范围。
- 需要联网下载依赖时先说明原因。
- 不允许未经确认执行删除、重置、清库、发布、部署命令。

## 推荐的项目准备

为了让 Codex 更快理解项目，可以准备：

- `README.md`: 项目用途、安装、启动、测试。
- `CONTRIBUTING.md`: 开发规范。
- `.env.example`: 环境变量示例，不包含真实密钥。
- 测试命令或 lint 命令。
- 关键业务术语说明。

如果项目没有这些文件，可以把“补齐项目 README”作为第一次练习。
