# 内部分享大纲：60 分钟 Workshop

## 分享主题

建议标题：

> Codex in Practice: 在中国大陆网络环境下安装、配置与日常开发工作流

## 目标听众

- 第一次使用 Codex 的开发、数据和 BI 同事。
- 想把 AI coding assistant 纳入日常工作流的团队成员。
- 需要了解使用边界和风险控制的 manager。

## 预期产出

参加者在 60 分钟后应能：

- 解释 Codex 和普通聊天机器人的区别。
- 在 Windows 上完成 Codex CLI 安装验证。
- 理解代理、npm、Git、登录之间的关系。
- 在项目目录中启动 Codex。
- 用 Codex 完成一次低风险修改并查看 diff。
- 知道哪些数据不能交给 Codex。

## 60 分钟结构

| 时间 | 内容 |
| --- | --- |
| 5 分钟 | Codex 是什么，适合哪些团队场景 |
| 10 分钟 | 账号、网络、系统环境准备 |
| 10 分钟 | Windows 安装演示 |
| 10 分钟 | 中国大陆网络与代理排查 |
| 15 分钟 | 项目目录实战：读代码、改 README、看 diff |

## 演示准备

提前准备：

- 一个不含敏感信息的 demo Git 仓库。
- 已安装 Git、Node.js、Codex CLI 的机器。
- 一个可用但合规的网络环境。
- 一份脱敏 README 或脚本文件。
- 常见错误截图或示例日志。

演示仓库建议包含：

- `README.md`
- `scripts/demo_export.py`
- `tests/test_demo_export.py`
- `.env.example`

## Demo 一：只读项目

提示词：

```text
请阅读这个 demo 项目的 README 和目录结构，概括项目用途、运行方式和主要文件。
暂时不要修改任何文件。
```

讲解点：

- Codex 会先建立项目上下文。
- 只读任务适合 onboarding。
- 第一次使用不要急着让它大改。

## Demo 二：小范围修改

提示词：

```text
请补充 README 的安装步骤。
只允许修改 README.md。
改完后总结你改了什么。
```

讲解点：

- 明确范围很重要。
- 修改后看 `git diff`。
- 人工确认表达是否符合团队风格。

## Demo 三：运行验证

提示词：

```text
请运行项目测试。如果失败，请先解释失败原因，不要直接修改代码。
```

讲解点：

- Codex 不只是生成文本，也能执行验证。
- 测试失败时应先分析，再修复。
- 涉及安装依赖或联网操作时要确认。

## 下一步练习

建议每位同事选择一个低风险任务：

- 给自己的项目补 README。
- 让 Codex 解释一个脚本。
- 给一个纯函数补测试。
- 对当前 git diff 做 code review。
- 把一次排障经验整理成 SOP。

完成后沉淀：

- 使用日期。
- Codex 版本。
- 使用场景。
- 提示词。
- 成功点。
- 出错点。
- 后续改进。
