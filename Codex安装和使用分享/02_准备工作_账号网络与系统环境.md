# 准备工作：账号、网络与系统环境

## 账号准备

使用 Codex 前，需要准备一种认证方式：

- ChatGPT 账号登录：适合个人或团队成员日常使用。
- OpenAI API Key：适合脚本化、CI 或需要明确按 API 计费的场景。
- 企业账号或团队账号：适合公司统一管理权限、账单和安全策略。

建议优先确认：

- 当前账号是否允许使用 Codex。
- 公司是否允许在该网络环境访问 OpenAI 服务。
- 是否有数据外发、日志留存、模型使用范围等内部要求。

## 系统环境

Windows 建议准备：

- Windows 10 或 Windows 11。
- PowerShell 5.1 或 PowerShell 7。
- Git for Windows。
- Node.js LTS。
- 一个可用于测试的 Git 项目目录。

检查命令：

```powershell
git --version
node -v
npm -v
```

如果命令无法识别，先安装对应工具，并重新打开 PowerShell。

## 推荐安装工具

Windows:

- Git for Windows: 用于 clone 项目、查看 diff、提交代码。
- Node.js LTS: 用于通过 npm 安装 Codex CLI。
- Visual Studio Code: 可选，方便查看文件和 Git diff。
- 公司批准的网络代理或安全网关: 用于稳定访问 npm、GitHub、OpenAI 等服务。

## 网络确认清单

在中国大陆网络环境下，常见失败点不是 Codex 本身，而是链路中的某一段不可达：

- npm registry 无法访问，导致安装失败。
- GitHub release 无法访问，导致安装脚本下载失败。
- OpenAI 登录页无法访问，导致 `codex login` 失败。
- 代理只配置给了浏览器，没有配置给 PowerShell、npm 或 Git。
- 公司代理使用自签证书，导致 TLS 证书校验失败。

建议先分别验证：

```powershell
npm ping
git ls-remote https://github.com/openai/codex.git
```

如果公司网络需要代理，请先阅读 [04_中国大陆网络环境配置.md](./04_中国大陆网络环境配置.md)。

## 安全边界

不要在教程、截图、聊天窗口或 Git 仓库中暴露：

- OpenAI API Key。
- ChatGPT 登录 token。
- 公司代理账号密码。
- 数据库连接串。
- 生产环境密钥。
- 包含客户个人信息或交易明细的样例数据。

如果必须演示真实项目，建议提前准备一个脱敏 demo 仓库。
