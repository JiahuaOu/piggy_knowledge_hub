# 快速认识：Codex 能做什么

## Codex 的定位

Codex 是 OpenAI 面向软件开发和自动化工作的 coding agent。它不是单纯的聊天机器人，而是可以在项目目录里阅读文件、理解上下文、修改代码、运行命令并汇报结果的开发助手。

常见入口包括：

- Codex CLI：在终端里进入项目目录后使用，适合开发者日常工作。
- IDE 扩展：在编辑器里辅助阅读和修改代码。
- Codex cloud / ChatGPT 里的 Codex 能力：适合处理任务、代码审查和远程环境中的开发工作。
- Codex 桌面端或应用集成：适合需要本地文件、浏览器、文档和命令行协作的场景。

团队内部分享建议以 **Codex CLI + Git 项目目录** 作为主线，因为它最容易建立可复现流程，也方便演示“修改前后对比”和“运行验证”。

## 它适合做的事情

代码类：

- 阅读项目结构并解释模块职责。
- 修改小 bug 或补充边界条件。
- 生成或补充单元测试。
- 根据错误日志定位问题。
- 做代码审查，指出潜在风险。
- 把重复脚本整理成可复用函数。

数据团队常见任务：

- 解释 SQL、Python、PowerShell 脚本。
- 帮忙整理 ETL 或报表逻辑说明。
- 生成数据字典、README、FAQ 草稿。
- 检查 SQL 里的口径不一致、字段误用和过滤条件风险。
- 把一次性分析脚本改成可维护版本。

文档和知识库任务：

- 把散乱笔记整理成教程。
- 按固定格式生成内部分享大纲。
- 将项目经验沉淀为 checklist。
- 对已有文档做查漏补缺。

## 它不适合直接做的事情

- 未经授权访问生产数据。
- 直接处理客户个人信息、商业敏感信息或密钥。
- 在没有人工确认的情况下执行破坏性命令。
- 替代代码 owner 做最终架构决策。
- 在不了解公司合规要求的情况下配置网络代理。

## 推荐心智模型

把 Codex 当成一个“能动手的搭档”：

- 你给目标、边界和验收标准。
- 它读取代码、提出方案、完成修改。
- 它运行检查，并说明做了什么。
- 你负责判断业务语义、合规边界和最终提交。

高质量指令通常包含：

- 背景：这个项目或文件是做什么的。
- 目标：希望完成什么具体结果。
- 范围：允许改哪些文件，不允许碰哪些文件。
- 验证：改完以后运行什么命令或检查什么结果。

示例：

```text
请阅读这个项目的 README 和 src 目录，帮我找出订单导出脚本失败的原因。
只允许修改 export_order.py 和对应测试。
改完后运行 pytest tests/test_export_order.py，并总结关键变更。
```
