## 分享定位

一句话目标：

> 让团队成员能在自己的项目目录里安全启动 Codex，并掌握“让它读代码、改代码、跑验证、解释结果”的基本工作流。

适合听众：

- 数据工程师、数据分析师、BI 开发同事。
- 需要维护脚本、SQL、文档或知识库的团队成员。
- 想了解 AI coding assistant 如何进入真实工作流的管理者。

## 建议先完成的最小闭环

第一次不要追求复杂自动化。建议先完成这个闭环：

1. 在 Windows 上安装 Codex CLI。
2. 解决当前网络环境下的登录和包下载问题。
3. 在一个 Git 项目目录中运行 `codex`。
4. 让 Codex 阅读 README 和一个小文件。
5. 让 Codex做一个低风险修改。
6. 让 Codex运行测试或检查命令。
7. 查看 `git diff`，确认变更是否符合预期。

这个闭环完成后，再进入更复杂的任务，例如批量重构、生成测试、编写分享材料、检查 Pull Request。

## 重要提醒

- 不要把 API Key、账号 token、客户数据、生产密钥粘贴到对话里。
- 公司网络、代理、VPN、证书相关配置应遵循 IT 和安全团队要求。
- Codex 可以帮你改文件和运行命令，但最终提交前仍需要人工 review。
- OpenAI 产品和命令可能更新，正式分享前请重新核对官方文档。
