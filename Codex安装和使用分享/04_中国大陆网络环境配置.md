# 中国大陆网络环境配置

## 基本原则

在中国大陆使用 Codex，网络问题通常来自三个环节：

- 安装阶段：需要访问安装脚本、npm 或 GitHub release。
- 登录阶段：需要访问 OpenAI / ChatGPT 登录服务。
- 使用阶段：需要访问 OpenAI API 或 Codex 后端服务。

请优先使用公司批准的网络代理、远程开发环境或安全网关。不要在公司设备上使用未经批准的工具或线路。

## 代理只配浏览器是不够的

很多同事遇到的问题是：浏览器能打开网页，但 PowerShell 里的安装命令失败。

原因是代理配置可能只对浏览器生效，没有传递给：

- PowerShell 当前进程。
- npm。
- Git。
- Node.js。
- Codex CLI。

## PowerShell 当前窗口代理

以下配置只对当前 PowerShell 窗口生效，适合临时测试：

```powershell
$env:HTTP_PROXY = "http://127.0.0.1:7890"
$env:HTTPS_PROXY = "http://127.0.0.1:7890"
$env:NO_PROXY = "localhost,127.0.0.1"
```

验证：

```powershell
npm ping
git ls-remote https://github.com/openai/codex.git
```

如果代理端口不是 `7890`，请改成你所在网络环境实际提供的地址和端口。

## 用户级代理环境变量

如果确认需要长期使用，可以设置用户级环境变量：

```powershell
[Environment]::SetEnvironmentVariable("HTTP_PROXY", "http://127.0.0.1:7890", "User")
[Environment]::SetEnvironmentVariable("HTTPS_PROXY", "http://127.0.0.1:7890", "User")
[Environment]::SetEnvironmentVariable("NO_PROXY", "localhost,127.0.0.1", "User")
```

设置后需要重新打开 PowerShell。

取消用户级代理：

```powershell
[Environment]::SetEnvironmentVariable("HTTP_PROXY", $null, "User")
[Environment]::SetEnvironmentVariable("HTTPS_PROXY", $null, "User")
[Environment]::SetEnvironmentVariable("NO_PROXY", $null, "User")
```

## npm 代理

如果 npm 单独失败，可以配置 npm：

```powershell
npm config set proxy http://127.0.0.1:7890
npm config set https-proxy http://127.0.0.1:7890
```

查看配置：

```powershell
npm config get proxy
npm config get https-proxy
npm config get registry
```

取消配置：

```powershell
npm config delete proxy
npm config delete https-proxy
```

## Git 代理

如果 GitHub 访问失败，可以配置 Git：

```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

取消配置：

```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```

## 公司代理和证书

如果公司代理会解密 HTTPS 流量，可能出现证书错误。常见报错包括：

- `self signed certificate`
- `unable to verify the first certificate`
- `certificate has expired`

推荐处理方式：

1. 向 IT 获取公司根证书安装方式。
2. 确认 Windows 系统证书是否已正确安装。
3. 按公司要求配置 Git、Node.js 或 npm 的证书信任。

不建议作为长期方案：

```powershell
npm config set strict-ssl false
```

这会降低安全性，只应在隔离测试环境里短时间排查问题，并且用完立即恢复：

```powershell
npm config set strict-ssl true
```

## 排查顺序

遇到网络问题时，按这个顺序定位：

1. 浏览器能否打开 OpenAI / ChatGPT / GitHub / npm 页面。
2. PowerShell 当前窗口是否设置了 `HTTP_PROXY` 和 `HTTPS_PROXY`。
3. `npm ping` 是否成功。
4. `git ls-remote https://github.com/openai/codex.git` 是否成功。
5. `codex login` 是否能打开登录流程。
6. Codex 进入项目后是否能正常响应。

每一步只改变一个变量，方便判断问题到底出在哪里。
