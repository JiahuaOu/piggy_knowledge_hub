# 常见问题与排错

## 1. 安装脚本下载失败

现象：

```text
The remote name could not be resolved
Connection timed out
```

处理：

1. 确认浏览器能否访问官方安装地址。
2. 确认 PowerShell 是否设置代理。
3. 尝试 npm 安装方式。
4. 如果公司网络限制下载，使用 IT 批准的内部软件源或手动下载 release。

## 2. npm install 很慢或失败

处理：

```powershell
npm config get registry
npm config get proxy
npm config get https-proxy
npm ping
```

如果是网络问题，先配置代理。不要一开始就删除缓存或重装 Node.js。

## 3. codex 命令找不到

检查：

```powershell
where.exe codex
npm config get prefix
$env:PATH
```

常见原因：

- npm 全局安装目录没有加入 `PATH`。
- 安装后没有重新打开 PowerShell。
- 使用了多个 Node.js 版本管理工具，路径指向不一致。

## 4. 登录流程卡住

可能原因：

- 浏览器走代理，CLI 没走代理。
- 本地回调端口被安全软件拦截。
- 公司网络拦截登录域名。
- 账号本身没有对应权限。

处理：

1. 重新打开 PowerShell 并设置代理。
2. 运行 `codex login`，按提示使用备用流程。
3. 如果登录页可以打开但回调失败，尝试复制设备码或手动授权方式。
4. 仍失败时，记录命令行报错和网络环境，交给 IT 或账号管理员确认。

## 5. Codex 能打开但响应失败

检查：

- 当前账号是否有效。
- API Key 是否过期或被撤销。
- 代理是否只对安装阶段生效，使用阶段失效。
- 公司网络是否对长连接或流式响应有限制。

可以先换一个最小问题测试：

```text
请回复一句 hello，不要读取文件。
```

如果最小问题也失败，多半是认证或网络问题。如果最小问题成功，但读项目失败，再检查项目权限和目录路径。

## 6. Codex 修改了不该改的文件

立即做：

```powershell
git status --short
git diff
```

如果不确定某个改动是否来自你，不要直接重置。先确认：

- 哪些是你原本已有的改动。
- 哪些是 Codex 本次新增的改动。
- 是否有未保存文件。

给 Codex 的后续指令：

```text
请只说明当前 git diff 里哪些改动是你刚才做的，不要继续修改文件。
```

## 7. 命令需要管理员权限

原则：

- 能用用户级安装就不要用管理员权限。
- 只有安装系统级工具、写入系统目录或修改全局 PATH 时才考虑管理员权限。
- 公司电脑上的管理员权限应遵循 IT 流程。

## 8. 证书错误

常见报错：

```text
self signed certificate
unable to verify the first certificate
```

处理：

1. 联系 IT 获取公司根证书配置方式。
2. 检查 Git、npm、Node.js 是否使用同一套证书信任。
3. 不要长期关闭 SSL 校验。

## 9. 中文路径或空格路径导致问题

如果某些工具对中文路径支持不好，可以先用英文路径验证：

```powershell
mkdir D:\work\codex-demo
cd D:\work\codex-demo
```

如果英文路径正常，再判断是否是路径编码、脚本引用或 shell 转义问题。

## 10. 如何准备求助信息

向同事、IT 或管理员求助时，建议提供：

- 操作系统版本。
- Codex 版本：`codex --version`。
- 安装方式：脚本、npm 或 release。
- 失败命令。
- 完整错误信息。
- 是否配置代理。
- 是否在公司网络、家用网络或远程环境。

不要提供：

- API Key。
- 登录 token。
- 公司代理密码。
- 真实客户数据。
