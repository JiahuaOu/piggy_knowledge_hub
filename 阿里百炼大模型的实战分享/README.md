## 项目定位

一句话目标：

> 建立一套基于阿里生态的大模型应用方法论，并用 Richemont 数据团队的真实业务场景验证可复用落地路径。

这个主题不建议只做成“百炼功能介绍”。更适合包装成数据团队的 AI enablement 项目：

- 对内：提升团队对大模型应用开发、RAG、智能体、评测治理的共同语言。
- 对业务：用低风险、高复用的知识问答和数据分析助手验证业务价值。
- 对管理层：展示数据团队在 AI adoption、业务赋能和技术治理上的主动性。

## 建议的首个 POC

优先做 **Richemont Data Knowledge Assistant**：

- 面向对象：数据团队、BI 使用者、业务分析师。
- 接入资料：数据字典、指标口径、报表说明、FAQ、团队 SOP。
- 核心问题：用户能否用自然语言快速理解指标、报表和数据口径。
- 技术覆盖：百炼模型调用、RAG 知识库、引用溯源、效果评测、权限与成本治理。

这个 POC 比“全自动 Agent”更容易成功，也更适合作为第一轮知识分享和管理层展示。
