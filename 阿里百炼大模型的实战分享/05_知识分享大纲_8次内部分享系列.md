# 知识分享大纲：8 次内部分享系列

## 系列定位

主题名称建议：

> From Knowledge to AI Assistant: Richemont 数据团队的阿里百炼实践路线

目标听众：

- 数据团队成员。
- BI 和数据分析相关同事。
- 对 AI 应用落地感兴趣的业务方。
- 需要了解 AI adoption 进展的管理层。

## 第 1 次：阿里百炼与企业 AI 应用总览

目标：

- 建立团队共同语言。
- 说明为什么数据团队适合推动 AI 应用实践。

内容：

- 大模型应用的基本形态。
- 阿里云百炼、DashScope、通义千问的关系。
- RAG、工作流、智能体、评测治理的概念。
- Richemont 数据团队可以切入的场景。

演示：

- 简单模型对话 Demo。
- 一个业务问题从“直接问模型”到“基于知识库回答”的差异。

## 第 2 次：模型调用与 API 接入实践

目标：

- 跑通模型调用。
- 建立团队代码模板。

内容：

- API Key 与环境变量管理。
- OpenAI 兼容接口思路。
- 模型参数解释。
- 流式输出、错误处理、重试。
- 成本和调用日志。

练习：

- 用 Python 或 Node.js 调用模型。
- 调整 temperature 对答案风格的影响。

## 第 3 次：RAG 知识库从 0 到 1

目标：

- 让团队理解知识库问答的基本机制。

内容：

- RAG 解决什么问题。
- 文档收集与清洗。
- 切片、向量化、召回、生成。
- 引用来源和拒答策略。

演示：

- 上传一组数据字典或报表说明。
- 让助手回答指标口径问题。

## 第 4 次：企业知识库质量优化

目标：

- 让知识库从“能答”走向“答得准”。

内容：

- chunk size 和 overlap。
- 元数据设计。
- TopK 和相似度阈值。
- Prompt 模板。
- 标准问题集和人工评分。

练习：

- 对同一批问题比较不同配置的效果。

## 第 5 次：Data Copilot 与数据分析场景

目标：

- 探索大模型如何辅助数据分析师和 BI 用户。

内容：

- 指标解释。
- 报表解读。
- SQL 草稿生成。
- 分析报告草稿。
- 与 DataWorks、MaxCompute、Hologres、Quick BI 的结合思路。

重点提醒：

- SQL 生成应先作为草稿。
- 生产数据访问必须走权限控制。

## 第 6 次：工作流与智能体

目标：

- 区分工作流和智能体的适用边界。

内容：

- 工作流适合可控流程。
- 智能体适合多步骤探索。
- 工具调用和 MCP 的概念。
- 人工确认节点。
- 审计日志。

案例：

- 活动复盘工作流。
- 数据质量排查助手。

## 第 7 次：安全、合规、成本与评测治理

目标：

- 让 AI 应用具备进入试点的条件。

内容：

- 数据分级。
- 个人信息和商业敏感信息处理。
- Prompt 日志风险。
- 成本估算和限流。
- 标准问题集。
- 幻觉率、引用命中率、拒答率。

输出：

- AI 应用上线前检查清单。

## 第 8 次：Richemont 场景 POC 展示

目标：

- 展示第一阶段成果。
- 收集反馈并争取下一阶段资源。

内容：

- Data Knowledge Assistant Demo。
- 真实问题演示。
- 评测结果。
- 风险控制。
- 下一阶段 Roadmap。

建议演示问题：

- 这个指标的定义是什么？
- 这个报表适合回答什么业务问题？
- 如果两个报表口径不一致，应该如何排查？
- 资料里没有的问题，助手如何拒答？
