# 知识介绍：阿里百炼与企业 AI 应用

## 1. 什么是阿里百炼

阿里云百炼是阿里云面向大模型应用开发的一站式平台，围绕通义千问等模型能力，提供模型调用、应用构建、知识库、工作流、智能体、插件或工具调用、评测和部署等能力。

对企业数据团队来说，百炼的价值不只是“能调大模型”，而是可以把大模型能力放进企业已有的数据、知识、流程和权限体系中。

## 2. 与阿里生态的关系

可以把阿里生态理解成四层：

| 层级 | 代表能力 | 数据团队关注点 |
| --- | --- | --- |
| 模型层 | 通义千问、Embedding、多模态模型 | 模型选择、质量、延迟、成本 |
| 平台层 | 百炼、DashScope API、OpenAI 兼容接口 | 应用开发、账号权限、API 接入 |
| 数据层 | DataWorks、MaxCompute、Hologres、AnalyticDB、OSS、Quick BI | 企业数据、报表、指标、权限 |
| 应用层 | RAG 知识助手、数据分析 Copilot、工作流、智能体 | 业务价值、可控性、上线治理 |

## 3. 百炼适合研究的核心能力

### 3.1 模型调用

研究重点：

- 如何通过 DashScope 或 OpenAI 兼容接口调用通义模型。
- 如何选择通用对话、长文本、多模态、Embedding 模型。
- 如何管理 API Key、地域、限流、重试、日志和成本。
- 如何把已有 OpenAI SDK 风格的代码迁移到阿里生态。

适合交付物：

- Python/Node.js 调用 Demo。
- 团队模型选择表。
- API Key 与环境变量管理规范。

### 3.2 RAG 知识库

RAG 适合解决企业知识更新快、内部资料多、模型本身不知道公司私有信息的问题。

关键流程：

1. 收集资料：PDF、Word、Markdown、网页、数据字典、报表说明。
2. 文档清洗：去重、去页眉页脚、修正表格、统一术语。
3. 切片与索引：设置 chunk size、overlap、元数据和向量化方式。
4. 检索召回：控制 TopK、相似度阈值、关键词过滤。
5. 生成答案：要求引用来源、限定回答范围、支持拒答。
6. 评测优化：通过标准问题集持续衡量准确率和幻觉率。

适合 Richemont 的知识内容：

- 数据字典和指标口径。
- Quick BI 报表说明。
- CRM、销售、库存、会员分析的常见问题。
- 团队 SOP、数据申请流程、权限申请流程。
- 品牌和产品培训资料中允许内部使用的内容。

### 3.3 工作流

工作流适合把可重复流程标准化，例如“输入活动名称，自动生成复盘报告草稿”。

典型场景：

- 活动复盘报告生成。
- BI 报表异常解释。
- 数据质量问题初步排查。
- 月报或周报摘要生成。
- 业务问题分流到正确的数据负责人。

设计原则：

- 可预测流程优先用工作流。
- 需要自由探索时再考虑智能体。
- 对关键业务结论保留人工确认节点。
- 对外部系统写操作设置审批或只读模式。

### 3.4 智能体与工具调用

智能体适合处理多步骤、目标导向、需要调用工具的任务，例如“根据用户问题判断是否需要查知识库、查指标字典、生成 SQL、总结结果”。

企业落地时应先限制边界：

- 明确智能体可以访问哪些工具。
- 明确工具是只读还是可写。
- 明确不能回答或必须升级人工的问题。
- 记录工具调用链路，便于审计和复盘。

### 3.5 评测与治理

这是数据团队最应该体现专业度的部分。

至少要评测：

- 答案准确率。
- 引用命中率。
- 幻觉率。
- 拒答正确率。
- 响应延迟。
- Token 成本。
- 用户满意度。
- 敏感信息处理是否合规。

没有评测的大模型应用，本质上还是 Demo；有评测和治理，才有进入生产环境的可能。

## 4. Richemont 数据团队的切入点

建议从低风险、高复用、高可见度的场景开始：

| 优先级 | 场景 | 价值 | 风险 |
| --- | --- | --- | --- |
| P0 | 数据知识助手 | 帮业务理解指标、报表、流程 | 低 |
| P0 | 报表解释助手 | 降低 BI 使用门槛 | 中 |
| P1 | 活动复盘助手 | 提升分析报告效率 | 中 |
| P1 | SQL 草稿生成 | 提升分析师效率 | 中到高 |
| P2 | 客户画像摘要 | 服务 CRM 和零售运营 | 高 |
| P2 | 商品图文理解 | 支持多模态探索 | 高 |

第一阶段不建议直接承诺“自动决策”或“自动操作业务系统”。更合适的定位是：辅助理解、辅助分析、辅助写作、辅助检索。

## 5. 对管理层的表达方式

可以这样总结：

> 我们计划围绕阿里云百炼建立企业级大模型应用最佳实践，从知识库、数据分析助手、工作流和评测治理四个方向推进。第一阶段选择数据知识助手作为 POC，验证大模型能否降低业务理解数据的门槛，并形成可复用的 AI 应用落地方法论。
